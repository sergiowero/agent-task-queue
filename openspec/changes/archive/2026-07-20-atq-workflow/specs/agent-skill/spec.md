## ADDED Requirements

### Requirement: Skill file defines agent protocol for ATQ CLI interaction
The system SHALL provide a skill file at `.opencode/skills/atq-workflow/SKILL.md` that teaches coding agents how to interact with the ATQ CLI. The skill SHALL be loaded by coding agents (OpenCode, Claude, Codex) when invoked by the user.

#### Scenario: Agent loads the skill
- **WHEN** a coding agent loads the atq-workflow skill
- **THEN** the agent understands the available CLI commands, the workflow protocol, and the guardrails

### Requirement: Agent identity section defines tool metadata
The skill SHALL include an identity section specifying toolName as "opencode", with version and model sourced from the agent's configuration. The sessionId SHALL reference the current session ID from the invoking tool (not generated by the agent). The role SHALL be specified by the user at skill invocation, defaulting to "senior".

#### Scenario: Agent constructs identity from skill
- **WHEN** an agent reads the identity section
- **THEN** the agent knows its toolName, can determine version and model, uses the current session ID, and knows its assigned role

### Requirement: Skill defines only claim and submit commands
The skill SHALL teach agents to use only `atq claim` and `atq submit-*` commands. The skill SHALL NOT teach agents to use `atq list` or `atq get`. The skill SHALL include guardrails explicitly prohibiting `atq list` and `atq get` usage.

#### Scenario: Agent follows command restrictions
- **WHEN** an agent reads the skill's command section and guardrails
- **THEN** the agent only calls `atq claim` and `atq submit-*` commands

### Requirement: Skill teaches phase intelligence via status mapping
The skill SHALL include a status-to-action mapping that tells the agent what work to perform based on the task status returned by `atq claim`. The mapping SHALL cover: planning, plan_changes_requested, coding, changes_requested, reviewing, and merging phases.

#### Scenario: Agent determines phase from status
- **WHEN** `atq claim` returns a task with status "planning"
- **THEN** the agent knows it must write an implementation plan

#### Scenario: Agent determines coding phase
- **WHEN** `atq claim` returns a task with status "coding"
- **THEN** the agent knows it must implement the code

#### Scenario: Agent determines review phase
- **WHEN** `atq claim` returns a task with status "reviewing"
- **THEN** the agent knows it must review the submitted code

### Requirement: All agent messages SHALL be in Markdown format
The skill SHALL require all `-m` message parameters to be written in Markdown format. The skill SHALL provide Markdown templates for each submission type (plan, code, review, merge) with appropriate headers, lists, and structure.

#### Scenario: Agent writes plan in Markdown
- **WHEN** an agent submits a plan via `atq submit-plan`
- **THEN** the `-m` message is formatted in Markdown with sections like "## Plan", "### Files", "### Decisions"

#### Scenario: Agent writes code summary in Markdown
- **WHEN** an agent submits code via `atq submit-code`
- **THEN** the `-m` message is formatted in Markdown with sections like "## Changes", "### Files modified", "### Testing"

### Requirement: Skill teaches worktree usage rules
The skill SHALL instruct agents to create a worktree only during coding, changes_requested, reviewing, and merging phases. The skill SHALL instruct agents NOT to create worktrees during planning or plan_changes_requested phases. The skill SHALL teach the worktree creation command: `git worktree add /tmp/atq-{task.id} {task.recommendedBranch}`.

#### Scenario: Agent creates worktree for coding
- **WHEN** an agent claims a task with status "coding" and no worktreePath exists
- **THEN** the agent creates a worktree at `/tmp/atq-{task.id}` before editing files

#### Scenario: Agent skips worktree for planning
- **WHEN** an agent claims a task with status "planning"
- **THEN** the agent does NOT create a worktree and works in the project directory

#### Scenario: Agent uses existing worktree for reviewing
- **WHEN** an agent claims a task with status "reviewing" and worktreePath is set
- **THEN** the agent uses the existing worktree at the path specified in worktreePath

### Requirement: Skill enforces git commit restrictions
The skill SHALL instruct agents NOT to run `git add`, `git commit`, or `git push` during planning, coding, or reviewing phases. The skill SHALL allow commits ONLY during the merging phase. The skill SHALL teach that commits happen during merge submission, not during code submission.

#### Scenario: Agent does not commit during coding
- **WHEN** an agent is working on a task with status "coding"
- **THEN** the agent does NOT run git add, git commit, or git push

#### Scenario: Agent commits during merge
- **WHEN** an agent is working on a task with status "merging"
- **THEN** the agent runs git add and git commit, then calls `atq submit-merge`

### Requirement: Skill teaches submission format for each command
The skill SHALL provide specific submission templates for each `atq submit-*` command. `submit-plan` SHALL include plan steps, files, and decisions. `submit-code` SHALL include changes, files modified, and testing. `submit-review` SHALL include findings, issues, and verdict. `submit-merge` SHALL include branch, commit, and summary.

#### Scenario: Agent follows submit-plan template
- **WHEN** an agent submits a plan
- **THEN** the message includes structured sections for plan steps, files to create/modify, and key decisions

#### Scenario: Agent follows submit-code template
- **WHEN** an agent submits code
- **THEN** the message includes structured sections for what changed, files modified, and how to verify

### Requirement: Skill teaches "no tasks available" handling
The skill SHALL instruct agents that when `atq claim` returns `success: false` with reason `no_tasks_available`, the agent SHALL stop and inform the user. The agent SHALL NOT retry indefinitely.

#### Scenario: Agent handles empty queue
- **WHEN** `atq claim` returns `{ "success": false, "reason": "no_tasks_available" }`
- **THEN** the agent stops and tells the user no tasks are available for their role
